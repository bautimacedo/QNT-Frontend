## PASO 4 — Crear/reescribir `src/views/PilotosView.vue`

Si el archivo existe pero es un placeholder o tiene contenido incompleto, reemplazarlo por completo.

### Estado del componente (`<script setup>`)

```
// Datos
pilotos              → ref([])
loading              → ref(false)
error                → ref('')
toast                → ref('')

// Filtros
searchText           → ref('')

// Modal de detalle
detailModal          → ref({ open: false, piloto: null })

// Permisos
user                 → inject('dashboardUser')
isAdmin              → computed(() => user.value?.authorities?.includes('ROLE_ADMIN'))
```

### Datos derivados (`computed`)

```
filteredPilotos →
  filtrar pilotos por searchText en: nombre, apellido, email
  (case-insensitive, búsqueda parcial)

cantidadTotal   → pilotos.length
cantidadFiltrada → filteredPilotos.length
hayFiltrosActivos → searchText.trim() !== ''
```

### Funciones principales

```
fetchPilotos()         // llama getPilotos(), asigna a pilotos, maneja error
openDetail(piloto)     // detailModal.open = true, detailModal.piloto = piloto
closeDetail()          // detailModal.open = false
clearFilters()         // searchText = ''
showToast(msg)         // asigna toast, limpia a los 4s
formatNombre(p)        // `${p.nombre} ${p.apellido || ''}`.trim()
getInitial(p)          // primera letra del nombre, uppercase
```

### Lógica de badges

**Badge `estado`:**
```
ACTIVO              → clase 'badge--green'   → texto 'Activo'
DESACTIVADO         → clase 'badge--red'     → texto 'Desactivado'
PENDIENTE_APROBACION → clase 'badge--gray'   → texto 'Pendiente'
```

**Badge `cmaVencimiento`:**
```
null               → texto 'Sin CMA'    → clase 'badge--gray'
fecha < hoy        → texto '[fecha]'    → clase 'badge--red'    (vencido)
fecha <= hoy+30    → texto '[fecha]'    → clase 'badge--yellow' (por vencer)
fecha > hoy+30     → texto '[fecha]'    → clase 'badge--green'  (vigente)
```
Para calcular: `new Date(cmaVencimiento)` vs `new Date()`.
Mostrar la fecha formateada como `dd/MM/yyyy` en los badges no-null.

**Avatar:**
```
tieneFotoPerfil = true  → mostrar initial del nombre en círculo con color de fondo
tieneFotoPerfil = false → ídem (la foto real no es accesible desde admin; usar siempre la initial)
```
El círculo con la initial del piloto es suficiente — no intentar cargar la imagen por URL.

## Notas para el programador

1. **La foto de perfil del piloto no es accesible desde admin** — `GET /mi-perfil/foto-perfil`
   requiere el token del propio piloto. Mostrar siempre la initial del nombre como avatar.
   No intentar cargar imágenes por URL construida con el ID.

2. **`cmaVencimiento` vs `licencias[].fechaVencimientoCma`** — son campos distintos.
   El `cmaVencimiento` a nivel raíz del piloto es el campo del usuario (sin CMA = campo del
   usuario desactualizado). Los `licencias[].fechaVencimientoCma` son de cada licencia ANAC.
   El badge de CMA en el listado usa `piloto.cmaVencimiento`.

3. **Ruta actual del router** — si la ruta `pilotos` ya apunta a `PilotosView.vue`, no hace
   falta tocar el router. Solo modificarlo si apunta a un placeholder.

4. **No agregar ruta de detalle `/pilotos/:id`** en esta versión — el detalle va en un modal
   dentro de `PilotosView.vue` para mantener consistencia con el resto de las vistas.
