# v0.14.0 — Perfil piloto: poder ver las imágenes del CMA y del Certificado de Idoneidad

**VERSIÓN:** v0.14.0  
**SLUG:** perfil-piloto-ver-imagenes-certificados  
**PRIORIDAD:** alta  
**DEPENDENCIAS:** Vista Perfil Piloto y detalle de piloto existentes

---

## Objetivo

Hoy en las pantallas de **perfil de piloto** las columnas "Img. CMA" y "Cert. Idoneidad" solo muestran un ✓ cuando hay imagen cargada, pero **no se puede ver la imagen**. La tarea es que el usuario pueda **abrir y ver** las imágenes del CMA y del Certificado de Idoneidad de cada licencia ANAC.

Hay dos vistas involucradas:

1. **PerfilPilotoView.vue** — El piloto (o un admin) ve sus propias licencias en `/perfil-piloto`. Ya existe el botón "Imágenes" que abre un modal; hay que asegurar que las imágenes se **muestren correctamente** en ese modal y que sea evidente que se pueden ver.
2. **PilotoPerfilDetalleView.vue** — Un admin ve el perfil de un piloto en `/pilotos/:id`. La tabla de licencias solo muestra ✓ en "Img. CMA" y "Cert. Idoneidad" **sin forma de ver las imágenes**. Hay que agregar la posibilidad de verlas.

## PASO 1 — PerfilPilotoView.vue (vista del piloto sobre sus propias licencias)

### 1.1 Revisar que el modal muestre las imágenes

En esta vista ya existe:

- Botón **"Imágenes"** por cada licencia que llama a `openLicImageModal(lic)`.
- Modal que carga `obtenerImagenCmaLicencia(lic.id)` y `obtenerImagenCertIdoneidad(lic.id)` y asigna las URLs al modal (`cmaUrl`, `certUrl`).

**Qué verificar/corregir:**

- Que el modal se abra al hacer clic en "Imágenes".
- Que mientras se cargan las imágenes se muestre un estado de carga (spinner) en cada sección.
- Que cuando la API devuelve el blob, se cree la URL con `URL.createObjectURL(blob)` y se asigne a `cmaUrl` / `certUrl`, y que en el template se muestre `<img :src="licImageModal.cmaUrl">` y `<img :src="licImageModal.certUrl">` para que las imágenes se **vean**.
- Que si una licencia no tiene imagen (404 o sin blob), se muestre un mensaje tipo "Sin imagen cargada" en esa sección en lugar de dejar el espacio vacío o roto.
- Que al cerrar el modal se revoquen las object URLs (`URL.revokeObjectURL`) para no tener fugas de memoria (si no está ya en `closeLicImageModal` o en `onUnmounted`).

Si algo de esto falla (por ejemplo las imágenes no se renderizan o el modal no muestra el `<img>`), corregirlo.

### 1.2 Hacer más evidente que se pueden ver las imágenes (opcional pero recomendado)

- En la columna "Imágenes" de la tabla, además de los badges "CMA ✓" y "Cert. ✓", se puede convertir cada badge en **link o botón** "Ver CMA" / "Ver Cert." que abra el mismo modal ya centrado en esa imagen (o un modal más chico que muestre solo esa imagen). Así el usuario no tiene que buscar el botón "Imágenes" para saber que puede ver.
- Alternativa: dejar el botón "Imágenes" pero con texto más claro, por ejemplo **"Ver imágenes"**, para que quede explícito que ahí se ven los certificados.

## PASO 2 — PilotoPerfilDetalleView.vue (vista admin: perfil de un piloto)

En esta vista el admin ve los datos de un piloto (nombre, email, datos de vuelo, licencias ANAC). La tabla de licencias tiene columnas "Img. CMA" y "Cert. Idoneidad" con solo un ✓ o "—", y **no hay forma de ver la imagen**.

### 2.1 Agregar acción "Ver imágenes"

- En la tabla de licencias ANAC, agregar una columna **"Acciones"** (o reutilizar espacio en las columnas actuales) con un botón o link **"Ver imágenes"** (o "Ver CMA / Cert.") por cada licencia.
- Solo habilitar el botón cuando la licencia tenga al menos una imagen (`lic.tieneImagenCma || lic.tieneImagenCertificadoIdoneidad`). Si no tiene ninguna, se puede mostrar "—" o el botón deshabilitado.

### 2.2 Modal para ver las imágenes

Al hacer clic en "Ver imágenes":

- Abrir un **modal** (mismo estilo que en PerfilPilotoView: dos secciones, "Imagen CMA" y "Certificado de Idoneidad").
- Para **cargar las imágenes** hace falta un endpoint del backend que permita a un **admin** obtener la imagen de una licencia de **otro usuario** (el piloto). Los endpoints actuales son:
  - `GET /mi-perfil/licencias/{id}/imagen-cma`
  - `GET /mi-perfil/licencias/{id}/imagen-certificado-idoneidad`  
  y devuelven las imágenes del **usuario autenticado**. Por tanto, para el perfil de otro piloto se necesita algo como:
  - `GET /usuarios/{pilotoId}/licencias-anac/{licenciaId}/imagen-cma`
  - `GET /usuarios/{pilotoId}/licencias-anac/{licenciaId}/imagen-certificado-idoneidad`  
  (o la convención que use el backend para "admin obtiene imagen de licencia de usuario X").

**Si el backend ya expone endpoints para que el admin obtenga las imágenes de un piloto:**

- En `src/api/` (por ejemplo en `usuarios.js` o en un módulo de licencias de usuario) agregar funciones del tipo:
  - `obtenerImagenCmaLicenciaPiloto(pilotoId, licenciaId)` → GET al endpoint correspondiente, devolver blob.
  - `obtenerImagenCertIdoneidadLicenciaPiloto(pilotoId, licenciaId)` → idem.
- En **PilotoPerfilDetalleView**, al abrir el modal de imágenes, usar `route.params.id` como `pilotoId` y `lic.id` como `licenciaId`, llamar a esas funciones y mostrar las imágenes en el modal igual que en PerfilPilotoView (con loading, `<img>` con la URL del blob, mensaje si no hay imagen).

**Si el backend aún no expone esos endpoints:**

- Implementar el modal y la UI (botón "Ver imágenes", modal con las dos secciones).
- En el modal, mostrar un mensaje tipo: "Para ver las imágenes de las licencias de otro piloto, el backend debe exponer endpoints de administración (ej. GET /usuarios/{pilotoId}/licencias-anac/{licenciaId}/imagen-cma)." y dejar los botones de carga o las secciones en estado "no disponible" hasta que existan.
- Documentar en el código o en un comentario qué endpoints se esperan.

### 2.3 Estilo del modal

- Reutilizar el mismo patrón que en PerfilPilotoView: modal con título "Imágenes de licencia", dos bloques (Imagen CMA / Certificado de Idoneidad), cada uno con posibilidad de spinner, imagen (`<img>`) o texto "Sin imagen cargada", y botón "Cerrar".
- En la vista admin no hace falta permitir **subir** imágenes (solo ver); si se reutiliza un componente, ocultar los controles de subida en este contexto.
