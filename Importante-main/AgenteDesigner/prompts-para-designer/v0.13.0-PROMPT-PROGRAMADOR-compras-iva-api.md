# v0.13.0 — Compras: IVA y alineación con la API

**VERSIÓN:** v0.13.0  
**SLUG:** compras-iva-api  
**PRIORIDAD:** alta  
**DEPENDENCIAS:** Vista de Compras existente (v0.8.x)

---

## Objetivo

Adaptar el frontend de Compras al nuevo modelo del backend: agregar soporte de **IVA** (tieneIva, ivaPorcentaje, subtotal) y asegurar que el request/response no use `numeroFactura`. El campo **importe** sigue siendo siempre el **total** de la compra; el backend calcula y devuelve **subtotal** cuando la compra tiene IVA.

## PASO 1 — Request de compra (crear / editar)

### 1.1 No enviar numeroFactura

En el body de **POST /compras** y **PUT /compras/{id}** no debe ir **numeroFactura**. Si en algún lugar del código se arma el body con `numeroFactura`, eliminarlo. Hoy en `buildBody()` no se envía; verificar que tampoco se agregue en esta tarea.

### 1.2 Incluir IVA en el body

En **buildBody()** agregar:

- **tieneIva** (boolean): valor del checkbox/toggle del formulario. Default `false` si no está definido.
- **ivaPorcentaje** (number, opcional): solo cuando `tieneIva === true`. Porcentaje entre 0 y 100 (ej. 21). No enviar o enviar `null` cuando `tieneIva` es false.

Ejemplo de lógica:

```js
body.tieneIva = !!f.tieneIva
if (body.tieneIva) {
  body.ivaPorcentaje = parseFloat(f.ivaPorcentaje)
} else {
  // no enviar ivaPorcentaje o enviar null según lo que espere el backend
  body.ivaPorcentaje = null
}
```

El resto de campos (proveedorId/proveedorNombre, fechaCompra, fechaFactura, importe, moneda, tipoCompra, metodoPago, tipoEquipo, descripcionEquipo, descripcion, observaciones, etc.) se mantienen según la documentación actual de la API.

## PASO 2 — Formulario de alta/edición de compra

### 2.1 Estado del formulario

En **FORM_DEFAULTS** (y al abrir el modal de edición con **openEdit**) agregar:

- **tieneIva**: `false` por defecto. Al editar: `compra.tieneIva ?? false`.
- **ivaPorcentaje**: `''` por defecto. Al editar: `compra.ivaPorcentaje != null ? String(compra.ivaPorcentaje) : ''` (ej. `'21'`).

### 2.2 Campos en el template del modal

- **Checkbox o toggle "Tiene IVA"**  
  - `v-model="formModal.tieneIva"`  
  - Label tipo: "Incluye IVA en el total" o "Tiene IVA".

- **Campo "Porcentaje IVA"** (solo visible cuando `formModal.tieneIva` es true):  
  - `v-model="formModal.ivaPorcentaje"`  
  - Tipo: `number`, step `0.01` o `1`, min `0`, max `100`.  
  - Placeholder ej. "21".  
  - Mensaje de error debajo si la validación falla.

Colocación sugerida: después del campo Importe o junto a Moneda/Tipo compra, para que quede claro que el importe es el total y el IVA es un desglose.

### 2.3 Validación

En **validateForm()**:

- Si **tieneIva** está marcado:
  - **ivaPorcentaje** es obligatorio.
  - Debe ser un número entre 0 y 100 (inclusive).
  - Mostrar mensaje de error en el campo (ej. "Ingresá el porcentaje de IVA (0–100).") si falta o está fuera de rango.

No cambiar la validación del importe: sigue siendo obligatorio y mayor a 0; el importe es siempre el **total**.

## PASO 3 — Respuesta de compra y detalle

### 3.1 Campos que devuelve la API

En el objeto **Compra** de la respuesta vienen (entre otros):

- **tieneIva** (boolean)
- **ivaPorcentaje** (number, ej. 21)
- **subtotal** (number): solo cuando tiene IVA; base imponible. Fórmula backend: `subtotal = importe / (1 + ivaPorcentaje/100)`.
- **usuarioAlta** (o usuarioAltaId / usuarioAltaEmail): usuario que dio de alta la compra (ya se usa en la vista).

No usar ni mostrar **numeroFactura** en ningún lugar (listado, detalle, formulario).

### 3.2 Modal de detalle de compra

Donde hoy se muestra solo el **Importe** (total):

- Seguir mostrando el **total** (`compra.importe`) con la moneda.
- Si **compra.tieneIva === true**:
  - Mostrar también:
    - **Subtotal:** `compra.subtotal` (formateado con la misma moneda).
    - **IVA (X%):** monto de IVA = `compra.importe - compra.subtotal` (o el valor que envíe el backend si viene explícito).
    - **Total:** `compra.importe`.
  - Formato sugerido:  
    `Subtotal $X + IVA 21% $Y = Total $Z`  
    usando `formatCurrency` para cada monto y la moneda de la compra.

Si el backend no envía `subtotal` cuando `tieneIva` es false, no mostrar la línea de subtotal/IVA en ese caso.

### 3.3 Listado (tabla)

- La columna **Importe** sigue mostrando el **total** (`c.importe`) con `formatCurrency(c.importe, c.moneda)`.
- Opcional: cuando `c.tieneIva === true`, mostrar un texto pequeño adicional tipo "(incl. IVA)" o un ícono/tooltip, sin cambiar el número mostrado.
