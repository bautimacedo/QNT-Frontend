## Problemas a resolver

### Bug 1 — Badge "Sin CMA" incorrecto

En `PilotosView.vue`, la función `cmaBadgeInfo` recibe solo `p.cmaVencimiento`
(campo a nivel de Usuario). Si ese campo es null pero el piloto tiene licencias ANAC
con `fechaVencimientoCma` cargada, el badge muestra "Sin CMA" de forma incorrecta.

**Causa:** `cmaBadgeInfo` ignora completamente `piloto.licencias`.

### Feature 2 — Página de perfil de piloto

El botón "Ver detalle" abre un modal. El usuario quiere que exista un botón
"Ver perfil" que navegue a una página dedicada `/pilotos/:id` con todos los
datos del piloto.

## PASO 1 — Fix badge CMA en `src/views/PilotosView.vue`

### 1.1 — Reemplazar la función `cmaBadgeInfo`

Actualmente:
```js
function cmaBadgeInfo(cmaVencimiento) {
  if (!cmaVencimiento) return { label: 'Sin CMA', cls: 'badge--gray' }
  ...
}
```

Reemplazar por dos funciones separadas:

```js
/**
 * Devuelve la fecha de CMA más relevante del piloto.
 * Prioridad: cmaVencimiento del usuario → fechaVencimientoCma de la licencia activa más reciente.
 */
function getCmaFechaEfectiva(piloto) {
  if (piloto.cmaVencimiento) return piloto.cmaVencimiento
  const fechas = (piloto.licencias || [])
    .filter(l => l.activo !== false && l.fechaVencimientoCma)
    .map(l => l.fechaVencimientoCma)
    .sort()
    .reverse()
  return fechas[0] || null
}

function cmaBadgeInfo(piloto) {
  const fecha = getCmaFechaEfectiva(piloto)
  if (!fecha) return { label: 'Sin CMA', cls: 'badge--gray' }
  const hoy = new Date()
  hoy.setHours(0, 0, 0, 0)
  const venc = new Date(fecha)
  const diffDias = Math.floor((venc - hoy) / (1000 * 60 * 60 * 24))
  const label = formatFecha(fecha)
  if (diffDias < 0) return { label, cls: 'badge--red' }
  if (diffDias <= 30) return { label, cls: 'badge--yellow' }
  return { label, cls: 'badge--green' }
}
```

### 1.2 — Actualizar todos los llamados a `cmaBadgeInfo`

`cmaBadgeInfo` ahora recibe el objeto `piloto` completo, no solo la fecha.

Buscar en el template todas las ocurrencias de `cmaBadgeInfo(...)` y reemplazar:

<script setup>
import { ref, computed, onMounted } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { getPilotos } from '../api'

const route = useRoute()
const router = useRouter()

const piloto = ref(null)
const loading = ref(false)
const error = ref('')

// Funciones utilitarias (mismas que PilotosView)
function formatNombre(p) {
  return `${p.nombre} ${p.apellido || ''}`.trim()
}

function getInitial(p) {
  return p.nombre ? p.nombre[0].toUpperCase() : '?'
}

function formatFecha(iso) {
  if (!iso) return '—'
  const [y, m, d] = iso.split('-')
  return `${d}/${m}/${y}`
}

function estadoBadgeClass(estado) {
  const map = { ACTIVO: 'badge--green', DESACTIVADO: 'badge--red', PENDIENTE_APROBACION: 'badge--gray' }
  return map[estado] || ''
}

function estadoBadgeLabel(estado) {
  const map = { ACTIVO: 'Activo', DESACTIVADO: 'Desactivado', PENDIENTE_APROBACION: 'Pendiente' }
  return map[estado] || estado
}

function getCmaFechaEfectiva(p) {
  if (p.cmaVencimiento) return p.cmaVencimiento
  const fechas = (p.licencias || [])
    .filter(l => l.activo !== false && l.fechaVencimientoCma)
    .map(l => l.fechaVencimientoCma)
    .sort()
    .reverse()
  return fechas[0] || null
}

function cmaBadgeInfo(p) {
  const fecha = getCmaFechaEfectiva(p)
  if (!fecha) return { label: 'Sin CMA', cls: 'badge--gray' }
  const hoy = new Date()
  hoy.setHours(0, 0, 0, 0)
  const venc = new Date(fecha)
  const diffDias = Math.floor((venc - hoy) / (1000 * 60 * 60 * 24))
  const label = formatFecha(fecha)
  if (diffDias < 0) return { label, cls: 'badge--red' }
  if (diffDias <= 30) return { label, cls: 'badge--yellow' }
  return { label, cls: 'badge--green' }
}

const cmaInfo = computed(() => piloto.value ? cmaBadgeInfo(piloto.value) : null)

async function load() {
  // 1. Intentar recuperar del history.state (viene de PilotosView)
  const fromState = history.state?.piloto
  if (fromState && String(fromState.id) === String(route.params.id)) {
    piloto.value = fromState
    return
  }
  // 2. Fallback: traer todos los pilotos y filtrar por ID (ej: navegación directa o refresh)
  loading.value = true
  error.value = ''
  try {
    const lista = await getPilotos()
    piloto.value = lista.find(p => String(p.id) === String(route.params.id)) || null
    if (!piloto.value) error.value = 'Piloto no encontrado.'
  } catch (e) {
    error.value = e.message || 'Error al cargar el perfil.'
  } finally {
    loading.value = false
  }
}

onMounted(load)
</script>

<template>

              {{ estadoBadgeLabel(piloto.estado) }}
            </span>
            <span class="badge" :class="cmaInfo.cls">

          </div>
        </div>
      </div>

        <h2 class="perfil-card-title">Datos de vuelo</h2>

          <div class="dato-item">
            <span class="dato-label">Horas de vuelo</span>
            <span class="dato-value">{{ piloto.horasVuelo ?? '—' }}</span>
          </div>
          <div class="dato-item">
            <span class="dato-label">Cantidad de vuelos</span>
            <span class="dato-value">{{ piloto.cantidadVuelos ?? '—' }}</span>
          </div>
          <div class="dato-item">
            <span class="dato-label">Venc. CMA (usuario)</span>

                  <span v-if="lic.fechaVencimientoCma" class="badge" :class="cmaBadgeInfo({ cmaVencimiento: lic.fechaVencimientoCma, licencias: [] }).cls">
                    {{ formatFecha(lic.fechaVencimientoCma) }}

## Notas para el programador

1. **`cmaBadgeInfo` ahora recibe el piloto completo**, no solo la fecha. Asegurarse de
   actualizar TODOS los llamados en el template de `PilotosView.vue` (hay dos: en la tabla
   y en el modal de detalle rápido).

2. **`history.state.piloto`** viene del `router.push({ state: { piloto } })` hecho en
   `verPerfil()`. En iOS/Safari y en algunos casos de hydration SSR puede no estar
   disponible — el fallback a `getPilotos()` cubre ese caso.

3. **El modal de "Detalle rápido"** se mantiene para ver la info sin salir de la lista.
   El botón "Ver perfil" lleva a la página completa. Ambos pueden coexistir.

4. **`PerfilPilotoView.vue`** (ya existente) es el perfil del usuario logueado (ROLE_PILOTO
   ve su propio perfil). `PilotoPerfilDetalleView.vue` es la vista de admin para ver
   el perfil de CUALQUIER piloto. Son cosas distintas — no confundir ni reemplazar.
