## PASO 1 — Ampliar `src/api/mi-perfil.js`

Agregar las siguientes funciones **al archivo existente** (no borrar las que ya están):

### CMA

```js
// GET /mi-perfil/cma → { cmaVencimiento, tieneImagenCma }
export async function getCma() {
  const res = await request('/mi-perfil/cma', { method: 'GET' })
  return json(res)
}

// PUT /mi-perfil/cma → body: { vencimiento: "yyyy-MM-dd" | null }
export async function actualizarCma(vencimiento) {
  const res = await request('/mi-perfil/cma', {
    method: 'PUT',
    body: { vencimiento },
  })
  return json(res)
}

// PUT /mi-perfil/cma/imagen → multipart/form-data, parte "file"
export async function subirImagenCma(file) {
  const formData = new FormData()
  formData.append('file', file)
  const res = await request('/mi-perfil/cma/imagen', {
    method: 'PUT',
    body: formData,
  })
  return json(res)
}

// GET /mi-perfil/cma/imagen → blob binario (image)
export async function obtenerImagenCma() {
  const res = await request('/mi-perfil/cma/imagen', { method: 'GET' })
  if (!res.ok) {
    if (res.status === 404) return null
    throw new Error('Error al obtener imagen CMA')
  }
  return res.blob()
}
```

### Licencias ANAC

```js
// GET /mi-perfil/licencias → array de licencias
export async function getMisLicencias() {
  const res = await request('/mi-perfil/licencias', { method: 'GET' })
  return json(res)
}

// POST /mi-perfil/licencias → body: CrearLicenciaMiPerfilRequest
export async function crearLicencia(body) {
  const res = await request('/mi-perfil/licencias', {
    method: 'POST',
    body,
  })
  return json(res)
}

// PUT /mi-perfil/licencias/{id} → body: ActualizarLicenciaMiPerfilRequest
export async function actualizarLicencia(id, body) {
  const res = await request(`/mi-perfil/licencias/${id}`, {
    method: 'PUT',
    body,
  })
  return json(res)
}

// DELETE /mi-perfil/licencias/{id}
export async function eliminarLicencia(id) {
  const res = await request(`/mi-perfil/licencias/${id}`, { method: 'DELETE' })
  if (!res.ok) {
    const text = await res.text()
    throw new Error(text || 'Error al eliminar licencia')
  }
  return null
}

// PUT /mi-perfil/licencias/{id}/imagen → multipart
export async function subirImagenLicencia(id, file) {
  const formData = new FormData()
  formData.append('file', file)
  const res = await request(`/mi-perfil/licencias/${id}/imagen`, {
    method: 'PUT',
    body: formData,
  })
  return json(res)
}

// GET /mi-perfil/licencias/{id}/imagen → blob
export async function obtenerImagenLicencia(id) {
  const res = await request(`/mi-perfil/licencias/${id}/imagen`, { method: 'GET' })
  if (!res.ok) {
    if (res.status === 404) return null
    throw new Error('Error al obtener imagen de licencia')
  }
  return res.blob()
}
```

### Lógica de visibilidad por rol

El usuario logueado puede tener distintos roles. Usar la info de `perfil.roles` (viene de `GET /mi-perfil`) para decidir qué secciones mostrar:

```js
const esPilotoOAdmin = computed(() => {
  if (!perfil.value?.roles) return false
  return perfil.value.roles.some(r =>
    r.codigo === 'ROLE_PILOTO' || r.codigo === 'ROLE_ADMIN'
  )
})
```

Las secciones de **CMA** y **Licencias ANAC** solo se renderizan si `esPilotoOAdmin` es true.

### Card [4]: CMA (Certificado Médico Aeronáutico)

**Al montar la sección (si esPilotoOAdmin):**
1. Llamar `getCma()` → obtener `{ cmaVencimiento, tieneImagenCma }`.
2. Si `tieneImagenCma` es true, llamar `obtenerImagenCma()` → crear URL.createObjectURL(blob) para mostrar la imagen.

**Editar fecha de vencimiento:**
- Input `type="date"` con el valor de `cmaVencimiento`.
- Botón "Guardar" → `actualizarCma(vencimiento)`.
- Éxito → toast "Vencimiento CMA actualizado".

**Imagen CMA:**
- **Si hay imagen:** Mostrar en un `<img>` con borde redondeado, tamaño contenido (~300px max). Debajo: botón "Cambiar imagen" (abre file input) y botón "Eliminar" (sube una imagen vacía o el backend soporta null — implementar como "reemplazar" subiendo una nueva).
- **Si no hay imagen:** Zona de upload: borde dashed, texto "Subir imagen del certificado CMA" + input file oculto que se activa con click.
- Al seleccionar archivo: `subirImagenCma(file)` → éxito → recargar imagen → toast "Imagen CMA cargada".
- Formato aceptado: imágenes (`accept="image/*"`).
- Límite: 10 MB (el backend limita; el front puede validar antes para mejor UX).

**Importante — manejo de imágenes binarias:**
El backend devuelve `application/octet-stream`. Para mostrar en `<img>`:
```js
const blob = await obtenerImagenCma()
if (blob) {
  cmaImageUrl.value = URL.createObjectURL(blob)
}
```
Recordar limpiar el object URL al desmontar (`onUnmounted` → `URL.revokeObjectURL()`).

## Notas para el programador

1. **No borrar el código existente de MiPerfilView.** Las 3 cards actuales (datos, contraseña, info vuelo) se mantienen. Las cards nuevas (CMA, Licencias) se agregan debajo de la card de info de vuelo.
2. **Imágenes binarias:** El backend devuelve `application/octet-stream`. Usar `res.blob()` (no `json()`) y `URL.createObjectURL(blob)` para el src del `<img>`. Limpiar con `URL.revokeObjectURL()` en `onUnmounted`.
3. **FormData para uploads:** El `request()` de `http.js` ya maneja FormData (no serializa a JSON si body es FormData). Solo hacer `new FormData(); formData.append('file', fileInput)`.
4. **Límite de 10 MB** por imagen (backend). El frontend puede validar `file.size > 10 * 1024 * 1024` antes de enviar y mostrar "La imagen no puede superar 10 MB".
5. **Fechas:** `cmaVencimiento` y `caducidad` vienen como `"yyyy-MM-dd"`. Para mostrar legible, formatear a `dd/MM/yyyy`. Para enviar, mantener ISO.
6. **Badge de vencimiento CMA:** Comparar con `new Date()`. Si `cmaVencimiento < hoy` → badge rojo "Vencido". Si `cmaVencimiento < hoy + 90 días` → badge amarillo "Próximo a vencer". Esto ayuda al piloto a renovar a tiempo.
7. **`GET /mi-perfil`** devuelve campo `tieneImagenCma` (boolean). Puede usarse para decidir si llamar a `obtenerImagenCma()` sin necesidad de un request adicional a `/mi-perfil/cma`.
8. **No existe endpoint DELETE para imagen CMA.** Para "eliminar" la imagen, la opción es subir una nueva. El botón debería decir "Cambiar imagen" más que "Eliminar".
9. **Misiones:** El backend aún no expone un endpoint de misiones asignadas al piloto. En la card de info de vuelo, mostrar `horasVuelo` y `cantidadVuelos` (vienen de `GET /mi-perfil`) con un placeholder "Misiones: próximamente" para cuando el backend lo implemente. No inventar un endpoint que no existe.
